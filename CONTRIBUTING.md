# Contributing to gh-pr-summarizer

We love your input! We want to make contributing to this project as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## We Develop with GitHub

We use GitHub to host code, to track issues and feature requests, as well as accept pull requests.

## We Use GitHub Actions, So All Code Changes Happen Through Pull Requests

Pull requests are the best way to propose changes to the codebase. We actively welcome your pull requests:

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Issue that pull request!

## Any contributions you make will be under the Apache Software License

In short, when you submit code changes, your submissions are understood to be under the
same [Apache License](https://github.com/simonmittag/gh-pr-summarizer/blob/main/LICENSE). Feel free to contact one of the project
maintainers if that's a concern.

## Report bugs using GitHub's [issues](https://github.com/simonmittag/gh-pr-summarizer/issues)

We use GitHub issues to track public bugs. Report a bug
by [opening a new issue](https://github.com/simonmittag/gh-pr-summarizer/issues/new/choose); it's that easy!

## Write bug reports with necessary detail, background, and sample code

Filing bugs right now is simple and should include either mention of failing tests or sample code to reproduce the issue.
